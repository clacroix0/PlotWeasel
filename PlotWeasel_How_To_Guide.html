Plot Weasel How-To Guide

Plain-language guide for using Plot Weasel Field Logger and Plot Weasel Desktop.

Current guide coverage:

  • Plot Weasel Field Logger v2.2.9, released July 16, 2026
  • Official Android APK: PlotWeaselFieldLogger_Ver2.2.9.apk
  • Plot Weasel Desktop v2.0.0, released June 4, 2026

Developed by the BIA Division of Forestry, Branch of Forest Inventory & Planning.

Guide updated: July 16, 2026

What this guide is for: This guide explains what each app does, how to move through the screens, what files to export, and what to do when something looks wrong. It is written for someone who has never used the apps before.

Quick Start

Use This App When You Need To Main Output
Plot Weasel Field Logger Collect plot, tree, and regeneration data on an Android APK, tablet, Surface, or field computer. Crew Package JSON, Site.csv, Tree.csv, Regen.csv, and PlotWeasel_upload.csv.
Plot Weasel Desktop Run forestry calculations from the exported Plot Weasel Desktop CSV or a regen CSV. Summary tables, graph files, audit files, stock tables, stand tables, crown ratio averages, and regen summaries.

Normal Workflow

  1. On Android field tablets, install and open the official Android APK. Confirm the app header shows Local only before field use.
  2. On Windows or Microsoft Surface devices, open the offline HTML version in Microsoft Edge.
  3. On an office computer or any field tablet, use Plot Weasel Field Logger to enter the project name and build the project species pick list.
  4. Export a Project Setup JSON file and copy it to each receiving crew tablet.
  5. On each receiving tablet, import the Project Setup file, then enter that tablet's Crew Name and Crew ID.
  6. Use Plot Weasel Field Logger on each tablet to collect field data.
  7. At the end of the day or exam, each crew exports a Crew Package.
  8. Each crew also exports a Backup JSON at the end of each day and before any app update, uninstall, or Clear Device Data action.
  9. Back at the office, import all Crew Packages into one secured copy of Field Logger.
  10. Export Site / Tree / Regen CSVs for reference and master records.
  11. Export Plot Weasel Desktop CSV for the Desktop calculator.
  12. Open Plot Weasel Desktop, upload the Plot Weasel Desktop CSV, choose the eco region, match the columns, and click Run.
  13. Review the results, audit, crown ratio averages, and graphs. Download the output files.

Security reminder: The apps run locally and do not need internet. Do not store sensitive tribal data in cloud folders unless your policy allows it. Keep tablets and exported files secured.

Part 1: Plot Weasel Field Logger

Plot Weasel Field Logger is the field data-entry app. It is designed for Android field tablets, Samsung tablets, Microsoft Surface devices, secured Windows computers, and offline use. It stores data locally on the device until the user exports it.

How To Open Field Logger

There are two supported ways to open Field Logger: the recommended Android APK path for Android tablets, and the offline HTML path for Windows or Microsoft Surface devices.

Recommended Android APK Path

  1. Copy PlotWeaselFieldLogger_Ver2.2.9.apk to the Android tablet.
  2. Open Files, My Files, or the approved file manager.
  3. Tap the APK file.
  4. If Android asks, allow installation from that source, such as Files, My Files, Chrome, Drive, or the approved file manager.
  5. Tap Install.
  6. Open Plot Weasel Field Logger from the app icon.
  7. Allow Location permission while using the app. Use Precise location if Android offers that option.
  8. Confirm the app header shows Local only.

Do not skip the storage check: Before real field use, create a test project, close the app completely, reopen it, and confirm the test project is still there. Restart the tablet and check again if possible.

Windows Or Microsoft Surface HTML Path

  1. Open PlotWeasel_Field_Logger.html in Microsoft Edge.
  2. If the computer asks what app to use, choose Microsoft Edge or the approved browser.

Best no-APK option: Use the normal Field Logger HTML or the single-file HTML version. In v2.2.9, both are self-contained with the app controls, style, icon, and FIA species list built into the file.

iPhone/iPad Note

For local files on iPhone or iPad, Safari and Chrome may treat the Field Logger HTML as a limited preview instead of a full app. If the page opens but Species, Trees, Regen, Add Plot, Aspect, or Slope Position do not respond, do not rely on that iPhone/iPad workflow for field collection until it has been separately tested.

Android Storage Mode Check

The app header may show a storage mode. Crews should understand the difference before entering real data.

ModeMeaningUse For Real Field Data?

Memory onlyData is only temporary and may disappear when the app closes, reloads, crashes, or Android clears it.No. Export Backup JSON immediately if important data was entered.

Android Updates And Uninstalling

  • Before installing an updated APK, export Backup JSON and confirm the backup file exists.
  • Do not uninstall the app before backing up data. Uninstalling can remove local project data.
  • APK installer files in Downloads can be deleted after installation. Deleting the installer file does not delete the installed app or its saved data.
  • If Android says App not installed during an update, stop and export Backup JSON before troubleshooting. The APK may have a different signing key, app identity, or version code.

MIT App Inventor Note

The MIT App Inventor WebViewer wrapper can open Field Logger, but the simple WebViewer build may show Memory only. Do not use a Memory only build for real field data. For v2.2.9 Android field testing, the Android Studio-built APK is the recommended APK route.

Main Screen Areas

  • Projects: Open a saved project, start a new project, or delete the active project from that device.
  • Field Session: Project name, crew name, and crew ID.
  • Plots: Add a new plot and choose the current plot.
  • Site tab: Site-level reference information.
  • Species tab: Choose the project species list from the FIA master species list.
  • Trees tab: Enter tree measurement records.
  • Regen tab: Enter regeneration records.
  • Review tab: Check warnings and errors before exporting.
  • Export / Merge tab: Export project setup, import project setup, export field files, backup, restore, and merge crew packages.

Switch Between Projects

Field Logger can keep more than one project in the same browser on the same tablet. Use the Projects panel on the left side of the app.

  • Open Project: Choose a saved project from the dropdown. The dropdown shows the project name and record counts.
  • New Project: Starts a blank project while keeping old projects saved on the device. It keeps the current Crew Name and Crew ID, but starts with blank plots, trees, regen, and project species.
  • Delete Project: Deletes only the active project from that device.
  • Project box: Renames the active project. Existing records follow that rename.
  • Clear Device Data: Removes all saved projects from that browser on that device.

Before opening note: A single shared HTML file or generic home-screen icon cannot show which saved project will open before launch. After the app opens, the Projects panel shows the active project, the browser tab title includes the active project name, and the app address includes that project's ID. To make a project-specific shortcut, open Field Logger, switch to the project, then add/bookmark that page and name the shortcut after the project.

Set Up A Field Session

  1. Enter the Project name.
  2. Enter the Crew Name.
  3. Enter the Crew ID, if your team uses one.
  4. Add the first plot number in the New plot # box and click Add.

Set Up The Species List

The app starts with an empty project species list. This is intentional. Each project can have its own species dropdown.

  1. Go to the Species tab.
  2. Search the FIA Master Species list by common name or FIA code.
  3. Select the species and click Add Species.
  4. Repeat until the project list has the species crews expect to use.
  5. Use Save Species List to save one editable species list on the device.
  6. Use Restore Saved List to reload that saved list later. Saving again overwrites the old saved copy.
  7. Use Clear Species List if you want to empty the project dropdown.

Adding species in the field: If a crew finds a species that was missed, go back to the Species tab, search the FIA list, and add it. The species will then appear in the Tree and Regen dropdowns. The Tree and Regen tabs no longer have a separate Add button next to Species.

Set Up Damage Agents

The Species tab also includes a Project Damage Agents list. Add project-specific damage agents there if the default choices are not enough. None and Broken Top stay available because they control app rules.

Set Up A Project And Share It

Use Project Setup when it is faster to build the project once on an office computer or field tablet and send the same setup to several tablets.

  1. Open Field Logger on the office computer or field tablet that will be the source setup device.
  2. Enter the Project name.
  3. Go to the Species tab and build the project species pick list from the FIA Master Species list.
  4. Add any project damage agents that crews should see.
  5. Go to Export / Merge.
  6. Click Export Project Setup. This creates a JSON setup file.
  7. Copy that JSON setup file to each receiving tablet using an approved local transfer method.
  8. On each receiving tablet, open Field Logger, go to Export / Merge, and choose Import Project Setup.
  9. After import, enter the tablet's Crew Name and Crew ID.

What Project Setup includes: Project name, project species pick list, one saved species list, and damage agents. It does not include plot records, tree records, regen records, Crew Name, or Crew ID.

Add Or Select A Plot

  • Use New plot # and Add to create a plot.
  • Use the Current Plot dropdown to switch between plots.
  • Any tree or regen record you save goes to the currently selected plot.

Enter Site Data

The Site tab is for plot-level reference information. Plot Weasel Desktop does not use Site.csv for calculations, but the forester may need the site data for review, master records, or project notes.

  • Fill in the fields that are available and useful.
  • Enter elevation as actual feet, such as 2500.
  • If slope is 5% or less, Aspect will only allow Level. If slope is greater than 5%, Level is not available.
  • At plot center, use Capture GPS 30 sec to collect device GPS fixes for about 30 seconds. The app averages the fixes and fills in whole-meter UTM Easting, UTM Northing, and UTM Zone.
  • If the device returns GPS altitude, the same capture fills Elevation as whole feet. Some tablets and browsers do not return altitude, so crews can still type elevation manually.
  • GPS Fix Count is the number of location readings collected during the capture. A higher number usually means the average had more readings to work from.
  • The tablet must have Location turned on, and the app or browser may ask for location permission. GPS capture does not need internet or cloud sync.
  • If GPS does not work or a crew has coordinates from another device, type the UTM values manually.
  • It is okay if some site fields are blank.
  • The app saves site information locally as you work.

Enter Tree Data

  1. Go to the Trees tab.
  2. Use the species filter if the dropdown is long.
  3. Select the species.
  4. For live trees, enter DBH and height. For dead trees, Actual Height is optional and can be used to record the dead tree’s actual height. It may be less than Height. For live trees, Actual Height remains available only when Broken Top is selected, and it must be greater than Height.
  5. Enter actual height if needed, cull, crown ratio, age, damage, decay class, live/dead status, and notes.
  6. Decay Class uses codes 1 through 5, can only be used when the tree status is Dead, and exports as DECAYCD. The dropdown shows the code descriptions.
  7. For DBH, values are stored and exported with one decimal place. Entering 10 becomes 10.0, entering 10.3 stays 10.3, and entering 103 becomes 10.3 when you leave the field.
  8. Click Save Tree Record.

Tree validation rules:

  • Live tree DBH is required and must be 5.0 inches or greater.
  • Live tree height is required and must be 8 feet or greater.
  • Dead tree DBH and height are optional. If entered, they must still be valid.
  • Species, DBH, and height are required for tree records that will be used in the Plot Weasel Desktop calculation export.
  • If Broken Top is selected on a live tree, both height and actual height must be entered.
  • Actual height must be greater than measured height for live broken-top trees.
  • Dead trees require Decay Class 1 through 5. DBH and height can be blank for dead trees. Actual Height is enabled and optional for dead trees. If entered, it must be greater than zero. Cull, Crown Ratio, and Age remain disabled for dead trees.
  • If cull is entered, it must be between 0 and 100.
  • If crown ratio is entered, it must be between 0 and 100.
  • If age is entered, it must be between 0 and 1000.

Decay Class Labels

Class 1Recently dead; bark and fine branches intact; wood hard.
Class 2Some bark loss; fine branches gone; wood firm.
Class 3Bark mostly gone; top often broken; wood starting to soften.
Class 4No bark; wood soft; form degrading.
Class 5Very soft, crumbling; snag collapsing or stump-like.

How Actual Height works: For normal live trees, crews enter Height and leave Actual Height alone. The app uses Height for both fields when exporting to Plot Weasel Desktop. Actual Height only turns on when a live tree is marked Broken Top. In that case, Height is the measured height to the broken top, and Actual Height is the estimated full height the tree would have had. Actual Height must be greater than Height. Dead trees do not use Actual Height.

Enter Regeneration Data

  1. Go to the Regen tab.
  2. Select a species from the project species list.
  3. Enter stem count.
  4. Select diameter class and height class if applicable. Use 0 inch for seedlings.
  5. Add damage notes if needed.
  6. Click Save Regen Record.

Regen validation rule: If Diameter Class is 2 inch or 4 inch, Height Class must be >5 ft.

Record A Null Plot

A null plot is a plot with no tree records. It still needs to be counted in the denominator for calculations.

  1. Add the plot number.
  2. Fill in any site information that is available.
  3. Do not add tree records for that plot.
  4. Export normally.

Field Logger automatically includes a plot-only null row in the Plot Weasel Desktop CSV and in Tree.csv.

Use The Review Tab

The Review tab helps catch issues before exporting.

  • Error means something should be fixed before exporting.
  • Warning means something may be okay, but should be checked.
  • Click a Review item to reopen the related plot, tree, or regen record when possible.
  • Tree and regen review items use labels like Plot 1 Tree 2 so the record is easier to find.
  • Use Refresh Review after changes if needed.

Merge Multiple Crews

  1. Each crew exports a Crew Package.
  2. Move those Crew Package files to the secured office computer.
  3. Open Field Logger on the office computer.
  4. Go to Export / Merge.
  5. Use Import Crew Package and select one or more crew package files.
  6. Review the merge summary.
  7. Use Save Site, Tree, Regen CSVs for the combined Site / Tree / Regen CSVs, then export the Plot Weasel Desktop CSV.

Move Work To Another Tablet

Use this when a tablet is down but you can still open Field Logger and export its data.

  1. On the original tablet, go to Export / Merge.
  2. Export a Crew Package.
  3. Move that Crew Package JSON to the replacement tablet using an approved local transfer method.
  4. On the replacement tablet, go to Export / Merge and choose Import Crew Package.
  5. If you import one Crew Package, Field Logger uses the package's Project Name, Crew Name, and Crew ID when the receiving tablet is blank. If the receiving tablet already has different values, the app asks before changing them.
  6. If you import multiple Crew Packages at once for office merging, Field Logger does not change the receiving device's Project Name, Crew Name, or Crew ID.

Part 2: Plot Weasel Desktop

Plot Weasel Desktop is the calculation and reporting app. It runs locally in a browser and does not need R, Python, internet, or admin rights.

How To Open Desktop

  1. Unzip the Plot Weasel Desktop release folder.
  2. Double-click Launch Plot Weasel Desktop v2.0.0.cmd, or open PlotWeasel_Desktop_v2.0.0.html.
  3. If PowerShell shortcuts are blocked on a government computer, use the launch CMD file or open the HTML file directly.

Desktop CSV Requirements

The exam CSV should include these columns, or columns that mean the same thing:

  • plot
  • spp, spcd, or species code
  • dbh
  • ht or height
  • actualht, optional
  • cull, optional
  • DECAYCD, optional dead-tree decay class code from 1 through 5
  • crown_ratio, optional
  • status, with values like live/dead or FIA-style 1/2 codes

Run A Desktop Calculation

  1. Click Upload exam CSV and choose the Plot Weasel Desktop CSV.
  2. Choose the eco region type, usually Province unless your project uses Division.
  3. Choose the eco region code.
  4. Check the column mapping. The app tries to match columns automatically, but the user should still confirm them.
  5. Set BAF and DBH class width if needed.
  6. Click Run.
  7. Read the status banner at the top of the results area. It should say the calculation completed.

Understanding Eco Regions In Desktop

Plot Weasel Desktop asks for an eco region because the NSVB coefficient tables can have region-specific rows. The selected eco region does not change the Logger records. It helps Desktop choose the best available coefficient row for the uploaded tree species.

  • Domain: the broadest ecological level. Desktop does not ask for this directly.
  • Division: a broad ecological subdivision. Use Division when the project is defined at that level or when a Province is not available.
  • Province: a more specific ecological unit inside a Division. Province is usually the better choice when the project has a known Bailey Province code.

The USDA Forest Service Bailey ecoregion reference describes the national ecoregion framework and provides map/list pages for Domains, Divisions, and Provinces. The Forest Service page notes that these ecoregions were adopted for ecosystem management as part of the National Hierarchical Framework of Ecological Units.

Reference: USDA Forest Service - Description of the Ecoregions of the United States. Useful lookup pages: Ecosystem Divisions and Ecosystem Provinces.

If a species is missing from Desktop outputs, check the Audit tab. A missing species can mean the uploaded CSV did not contain that species as a tree record, or that Desktop could not find the needed NSVB coefficient for the selected eco region and species.

Download Desktop Outputs

  • Use Download CSVs to download the full output zip.
  • Use Download table inside Stock, Stand, or Crown Ratio to download that specific table.
  • Use Download audit inside Audit to save the audit text file.
  • The full output zip includes CSV tables, audit text, and SVG graph files.

Use The Regen Tab In Desktop

The Regen tab runs the Tiny Weasel regeneration workflow.

  1. Go to the Regen tab.
  2. Upload a regen CSV.
  3. Confirm the expansion factor. The default is 250.
  4. Click Run Regen.
  5. Use the graph tabs to view Species TPA or Species x Height.
  6. Click Download Regen Outputs to save regen tables, graph SVGs, and the regen audit.

The regen CSV should include:

  • plot
  • spp or species
  • Stem Count
  • Diameter Class
  • Height Class

Null Plots In Desktop

Desktop includes null plots when the uploaded CSV contains a plot row with tree fields left blank. Field Logger creates those placeholder rows automatically for plots with no tree records.

The Audit tab reports how many null plots were detected.

Plain-English File Glossary

  • Android APK: The installable Android app file. For this release, the official APK is PlotWeaselFieldLogger_Ver2.2.9.apk.
  • APK installer file: The file tapped to install the app. It can be deleted from Downloads after installation without deleting the installed app or app data.
  • Local only: Expected app storage mode. Data is stored locally on the device until exported or cleared.
  • Memory only: Temporary storage mode. Do not use for real field collection because data may disappear when the app closes or reloads.
  • Project Setup: A JSON file used to copy project settings from one Field Logger device to crew tablets. It does not contain field records.
  • Crew Package: A JSON file used to move one crew's field data into another copy of Field Logger for merging or tablet replacement. It includes field records plus Crew Name and Crew ID.
  • PlotWeasel_upload.csv: The CSV that goes into Plot Weasel Desktop.
  • Site.csv: Site-level reference information, including UTM/GPS fields when captured. Desktop does not use this for calculations.
  • Tree.csv: Tree-level data for reference and master records. It includes plot-only null rows for plots with no tree records.
  • Regen.csv: Regeneration records for reference, master records, or Tiny Weasel regen calculations.
  • Review.txt: Review messages from Field Logger.
  • data_audit.txt: Desktop audit details for the calculation run.
  • SVG graph files: Browser-generated graph files included in Desktop output zips.

Guide Maintenance Checklist

Use this checklist every time the apps are updated. The guide should be reviewed before each new release zip is shared.

  • Update the Desktop version number and release date.
  • Update the Field Logger version number and release date.
  • Add any new buttons, tabs, validation rules, or exports.
  • Remove instructions for features that were removed.
  • Check the normal workflow still matches the current app behavior.
  • Check the troubleshooting section for new common questions.
  • Open this HTML file in Microsoft Word and confirm the headings, tables, and bullets look clean.
  • Save a fresh copy of the guide in the output folder with the current update date.
  • Update the official APK filename and SHA-256 checksum in README, release notes, and GitHub Releases.
  • Confirm the Android APK installs and shows Local only before sharing it with crews.
  • Confirm Backup JSON and Plot Weasel Desktop CSV export work on the target tablet.
  • Do not include .jks signing keys, keystore passwords, or key passwords in GitHub or release zip files.
